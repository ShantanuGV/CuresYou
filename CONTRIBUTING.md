# Contributing to CurseYou

First of all, thank you for your interest in contributing to **CurseYou**.

CurseYou is a procedural graphics engine focused on computational geometry, optimization, and image composition. We welcome contributions that improve the project's quality, performance, documentation, and maintainability.

---

# Ways to Contribute

You can contribute by:

* Fixing bugs
* Improving documentation
* Optimizing algorithms
* Improving rendering quality
* Adding unit tests
* Refactoring existing modules
* Expanding the dataset
* Suggesting new features

---

# Development Workflow

1. Fork the repository.
2. Create a new feature branch.

```bash
git checkout -b feature/my-feature
```

3. Make your changes.

4. Run all tests before committing.

5. Commit using meaningful commit messages.

```text
feat: add bridge overlap optimization

fix: improve affine alignment

docs: update architecture documentation
```

6. Push your branch.

```bash
git push origin feature/my-feature
```

7. Open a Pull Request.

---

# Coding Standards

## Python

* Follow PEP 8
* Use type hints
* Write descriptive docstrings
* Avoid global variables
* Keep functions focused on a single responsibility

---

# Pull Request Checklist

Before submitting a Pull Request, ensure that:

* [ ] Code builds successfully
* [ ] Existing functionality remains unchanged
* [ ] New functionality includes documentation
* [ ] Tests pass
* [ ] No unnecessary files are included
* [ ] Commit messages are descriptive

---

# Reporting Bugs

When opening an issue, please include:

* Operating System
* Python version
* Error logs
* Steps to reproduce
* Expected behavior
* Screenshots (if applicable)

---

# Feature Requests

Feature requests should include:

* Problem statement
* Proposed solution
* Alternative approaches
* Expected benefits
* Possible drawbacks

---

# Code of Conduct

By participating in this project, you agree to be respectful, constructive, and collaborative.

Thank you for helping improve CurseYou.
